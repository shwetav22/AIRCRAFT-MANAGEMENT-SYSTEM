# AIRCRAFT-MANAGEMENT-SYSTEM
# Flight Management SQL Project — Description
A relational database system that models airline flight operations, built around five linked tables: aircraft, airports, flights, passengers, and bookings. The schema uses primary and foreign keys to connect flights to aircraft, and bookings to both flights and passengers, enabling realistic queries across the network of relationships.
The project includes sample data — 3 aircraft, 5 Indian airports (Delhi, Mumbai, Bangalore, Hyderabad, Pune), 15 flights, 15 passengers, and 15 bookings — along with a set of analytical queries covering:

Passenger lists for a given flight
Flight occupancy rate (seats booked vs. total capacity)
Busiest arrival airports
Revenue generated per flight
Fully booked flights
Frequent flyers (passengers with more than 5 bookings)
Flights departing from a specific city on a specific date
Most frequently used aircraft
Average fare by route
Underbooked flights (occupancy below 50%)
